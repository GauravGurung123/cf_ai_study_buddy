# Contributing to AI Study Buddy

Thank you for your interest in contributing! feedback and suggestions are welcome.

## Code of Conduct

Please be respectful and constructive in all interactions.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists
2. Create a detailed issue report including:
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Environment details

### Suggesting Features

Feature suggestions are welcome! Please include:
- Use case description
- Expected behavior
- Why it would be valuable

### Code Contributions

Direct code contributions may not be accepted during the review period. However, you can:

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request with detailed description

## Development Setup

See [QUICKSTART.md](./QUICKSTART.md) for setup instructions.

## Coding Standards

### TypeScript
- Use TypeScript for all new code
- Follow existing code style
- Add types, avoid `any`
- Document complex logic

### Code Style
- Use Prettier for formatting
- Follow ESLint rules
- Write clear, descriptive variable names
- Add comments for complex logic

### Commits
- Write clear commit messages
- Use conventional commits format:
  - `feat:` for new features
  - `fix:` for bug fixes
  - `docs:` for documentation
  - `refactor:` for refactoring
  - `test:` for tests

### Testing
- Test your changes locally
- Ensure TypeScript compiles without errors
- Test in development environment before submitting
